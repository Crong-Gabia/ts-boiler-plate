# MVP Rules (Checklist)

이 문서는 엔지니어 합의를 빠르게 만들기 위한 "최소 룰"입니다.

## MUST (위반 시 PR block)
1) 타입 오류 금지(타입 suppress 금지 포함)
2) 테스트 실패 금지(최소 health 시나리오 포함)
3) 비밀(토큰/키) 커밋/로그 노출 금지
4) 빈 catch 블록 금지(원인 삼키기 금지)
5) 에러 응답은 표준 포맷(코드/메시지/traceId 등) 사용
6) 인증이 필요한 API는 무조건 401/403 규칙 준수
7) 입력 검증 누락 금지(최소 schema/validation)
8) 마이그레이션/스키마 변경은 롤백 플랜 포함
9) 변경 영향 범위에 맞는 테스트 실행/기록(검증 방법)
10) PR 본문에 변경 요약/검증 방법/리스크/롤백 포함

## SHOULD (경고 + 근거 요구)
1) 에러 코드 체계는 도메인별로 분리
2) DTO/JSON 필드 네이밍 규칙을 프로젝트 전체에 통일
3) 로그에 requestId/traceId를 포함
4) 외부 의존성(시간/랜덤/네트워크)은 테스트에서 통제
5) PR은 가능한 작게(리뷰 30분 내)
6) API는 리소스 기반 경로 설계(동사 남발 금지)
7) 권한 모델은 문서화(RBAC/ABAC 중 선택 근거)
8) 성능에 민감한 경로는 기준(SLA/쿼리 수) 기록
9) 취약한 패턴(eval, 임의 SQL string 등) 금지
10) 비동기 작업/큐는 idempotency 키 설계

## NICE
- 포맷/임포트 정렬 자동화
- 문서/주석 보강
