# 테스트 가이드 (General)

이 문서는 특정 테스트 프레임워크에 종속되지 않는 **일반 테스트 작성 규칙**입니다.

## 목표
- 회귀(regression) 방지
- 변경 안전성 확보
- 개발 속도 향상(자신감)

## 테스트 레벨
- 단위(Unit): 함수/클래스/모듈
- 통합(Integration): DB/외부 연동 경계 포함
- E2E: 사용자 플로우

## 기본 규칙
- 1 테스트 = 1 행동(behavior)
- 외부 의존성(네트워크/시간/랜덤)은 통제(mock/fake)
- flaky 테스트 금지
- 실패 메시지가 원인을 설명하도록(assert 메시지/구조)

## 커버리지
- 수치 목표는 팀 합의(예: 핵심 로직 80%+)
- 커버리지 수치보다 "중요 로직" 우선

## 실행/검증
- PR 전: 변경 영향 범위에 맞는 테스트 실행
- 머지 전: CI에서 전체 테스트 통과
